# T0.1.9 Model Spike — bge-small-en-v1.5 re-validation

**Status:** Closed. Recommendation (a) — confirm bge-small-en-v1.5 holds, no change. Reconciled across two independent passes (Shahbaz + Claude); both landed (a).
**Date:** 2026-05-03
**Scope:** Re-validate the bge-small-en-v1.5 lock from BRD §5.3 + ADR-020 before any further model-touching work. Triggered by the natural prompt of CI fixture friction (Step 2a/2b) re-encountering the assumption.
**Methodology:** Web research only. No code, no `Cargo.toml` edits, no fixture downloads, no SHA changes. Findings doc only.
**NOT in scope:** Production bundling (locked by BRD §5.3 + ADR-019, stands regardless of model). CI fixture strategy (`actions/cache@v4` is the answer regardless of model).

## 1. License verification — bge-small-en-v1.5

**License:** **MIT**.
- Source: HuggingFace model card <https://huggingface.co/BAAI/bge-small-en-v1.5>.
- Verbatim: *"FlagEmbedding is licensed under the [MIT License](https://github.com/FlagOpen/FlagEmbedding/blob/master/LICENSE). The released models can be used for commercial purposes free of charge."*
- The author's statement explicitly extends MIT to **model weights**, not only the FlagEmbedding code repository.

**Commercial closed-source distribution:** ✅ permitted unconditionally.

**Ethical-use / non-commercial / responsible-AI clauses:** ✅ none. No RAIL clauses, no "responsible use" riders, no field-of-use restrictions.

**Attribution requirement:** Soft (academic-norm citation, not legal).
- Citation requested in BibTeX form (paper: *"C-Pack: Packaged Resources To Advance General Chinese Embedding"*).
- This is academic norm, not an MIT-license attribution clause.
- MIT's standard *"include this permission notice in all copies"* requirement is satisfied by including the MIT LICENSE text in the distributed binary; vault-tauri's resource-bundling already accommodates this.

**Conclusion: license is clean for commercial closed-source distribution. No concrete reason-to-switch from licensing.**

## 2. Alternatives survey

Five candidates surveyed alongside the current choice. All six pass the binary license pass/fail (commercial closed-source clean). Differences emerge at integration cost, ONNX-officiality, and retrieval signal.

| # | Model | License | Dim | Max seq | Params | Official ONNX | Prefix req | Retrieval signal |
|---|---|---|---|---|---|---|---|---|
| **0** | **bge-small-en-v1.5** *(current)* | MIT, commercial-OK explicit | 384 | 512 | 33.4M | ✅ BAAI's own repo at `onnx/model.onnx` | None | **51.68 NDCG@10** |
| 1 | all-MiniLM-L6-v2 | Apache-2.0 | 384 | 256 (trained at 128) | 22.7M | Tag listed; not confirmed first-party | None | Significantly lower (~50 ArguAna alone) |
| 2 | gte-small (Alibaba) | MIT | 384 | 512 | 33.4M | Tag listed; not confirmed first-party | None | 49.46 retrieval avg |
| 3 | e5-small-v2 (Microsoft) | MIT | 384 | 512 | ~33M | Tag listed; not confirmed first-party | **Mandatory** `query: ` / `passage: ` | Comparable class |
| 4 | snowflake-arctic-embed-s | Apache-2.0 | 384 | 512 | 33M | Tag listed; not confirmed first-party | **Likely inherited from e5-small base** | **51.98 NDCG@10** |
| 5 | nomic-embed-text-v1.5 | Apache-2.0 | 768 (MRL→64-512) | 8192 | ~100M | Not officially provided | **Mandatory** `search_query:` / `search_document:` | 62.28 full MTEB avg (different size class) |

### Key per-candidate observations

**bge-small-en-v1.5 (current):** First-party ONNX export from BAAI in the same repo as the safetensors. No prefix injection required. Standard BERT tokenizer.

**all-MiniLM-L6-v2:** Smallest of the lot at 22.7M params. Trained at sequence-length 128 (auto-truncated to 256 at inference); meaningfully shorter context than the 512 of the others. Retrieval signal materially weaker.

**gte-small:** MTEB averages comparable but retrieval-specific score lower than bge-small. Same 33.4M-param class. ONNX availability listed in tags but not confirmed as first-party from Alibaba DAMO.

**e5-small-v2:** Has **mandatory text prefix requirement** — `query: ` for queries, `passage: ` for documents. Omitting prefixes degrades performance per the model card. Would require a `BgeSmallProvider`-equivalent API redesign in `vault-embedding` (prefix injection at write-time and query-time, with role tagging).

**snowflake-arctic-embed-s:** Edges bge-small by 0.30 NDCG@10 retrieval — **within measurement noise**. Built on `intfloat/e5-small-unsupervised` (the e5 family's base), which **likely inherits e5's mandatory prefix requirement** (verification needed before any switch). Apache-2.0 vs MIT is technically more permissive but not load-bearing for our distribution model.

**nomic-embed-text-v1.5:** Different size class (~3× params, ~5× disk). 8192-token context with Matryoshka truncation to 384/512 dim is genuinely useful for the long-content connectors at T0.2.x. Mandatory prefix requirement (`search_query:` / `search_document:`) is even stricter than e5's. **Re-evaluate at T0.2.x as the natural candidate when the 512-token ceiling becomes concrete** (see Forward Concerns §6).

## 3. Decision matrix

Hard requirement first, then weighted criteria.

### Hard requirement (binary pass/fail)

**License clean for commercial closed-source distribution:** all six candidates pass. Not a discriminator.

### Weighted criteria

| Criterion | bge-small | MiniLM-L6 | gte-small | e5-small-v2 | arctic-s | nomic-v1.5 |
|---|---|---|---|---|---|---|
| **Retrieval (NDCG@10 / MTEB-R proxy)** | **51.68** | ~50 | 49.46 | ~similar | **51.98** | n/a peer (62.28 full-MTEB at different size class) |
| **Inference latency feasibility (BRD §5.3 100ms M1)** | ✅ measured 12.3 ms median | ✅ smaller, faster | ✅ same class | ✅ same class | ✅ same class | ⚠ ~3× larger, may break ceiling |
| **Disk size (installer fatness)** | 130 MB ONNX (baseline) | ~90 MB | ~130 MB | ~130 MB | ~130 MB | ~550 MB MRL-truncatable |
| **ONNX export maturity** | ✅ **first-party (BAAI)** | Tag listed, not confirmed | Tag listed, not confirmed | Tag listed, not confirmed | Tag listed, not confirmed | Not officially provided |
| **Prefix-injection requirement** | None | None | None | **Mandatory query/passage** | **Likely inherits from e5 base** | **Mandatory search_query/search_document** |
| **API-surface drop-in for `BgeSmallProvider`** | ✅ baseline | ✅ drop-in | ✅ drop-in | ❌ requires API redesign | ❌ likely requires API redesign | ❌ requires API redesign + size class |

### What's load-bearing in this matrix

- **Official ONNX from the model author** is the single strongest differentiator for our trust posture per ADR-019's signed-binary chain. A community ONNX conversion is a different trust model: we can't independently verify the conversion preserved the trained weights without re-running the export ourselves. bge-small is the **only** candidate where this is unambiguous.
- **Prefix-injection requirements** are not a footnote — they require a concrete `BgeSmallProvider`-API change (role tagging, prefix injection on both write and query paths, schema for "what role is this text playing?") and would propagate into `vault-retrieval`'s pipeline.
- **The 0.30 NDCG@10 gap to arctic-s is within measurement noise.** Not a decision-grade signal on its own, especially against the cost above.

## 4. Recommendation

**(a) Confirm bge-small-en-v1.5 holds. No change.** All locked artefacts (ADR-019, ADR-020, ADR-022, BRD §5.3, `integrity.rs` SHA constants) stand without amendment.

### Reasoning

1. **License clean** — MIT, commercial-OK explicit, no clauses to navigate. No concrete reason-to-switch surfaced.
2. **Retrieval at top of class** — 51.68 NDCG@10 is at the head of the 384-dim/33M-param tier; only arctic-s edges it by 0.30, within noise.
3. **First-party ONNX export** — BAAI publishes `onnx/model.onnx` directly; no other surveyed candidate has this verified. Load-bearing for ADR-019's signed-binary chain trust posture.
4. **No integration prefix tax** — `BgeSmallProvider`'s current API surface is a clean fit; e5-family alternatives (e5-small-v2, arctic-s, nomic) would require API redesign for prefix injection.
5. **Disk size at the implicit BRD §5.3 budget** — 130 MB ONNX matches what the 100 ms M1 latency ceiling implicitly accepts.

### Forward-compat-pull test on the only credible alternative

**Is "snowflake-arctic-embed-s edges bge-small by 0.30 NDCG@10" a concrete reason to switch?**

No. It's hypothetical *("might be slightly better")* with three caveats:
- **Score gap is within measurement noise** — 0.30 NDCG@10 is below the threshold of decision-grade signal.
- **arctic-s likely inherits e5-base's prefix requirement.** Built on `intfloat/e5-small-unsupervised`. Would need verification before any switch decision; if confirmed, requires `BgeSmallProvider` API redesign.
- **ONNX availability not confirmed first-party.** Switching weakens ADR-019's signed-binary chain unless Snowflake's first-party export is verified.

**Defer per the standing rule.** If a future swap is ever evaluated, prefix-requirement compatibility with `BgeSmallProvider`'s API surface is a **first-class evaluation criterion**, not a footnote.

## 6. Forward concerns

Three items to log into HANDOFF.md tech-debt as watch-points for downstream phases. **None are present-day switch triggers.**

### 6.1 — 512-token context ceiling vs T0.2.x connector ingestion

**Watch-point:** bge-small-en-v1.5's 512-token max sequence becomes a concrete constraint at T0.2.x connector ingestion (BRD §5.9). Long emails, PDFs, transcripts, and conversation logs may routinely exceed 512 tokens. The current naïve truncation strategy (right-truncate at 512 tokens) loses tail content; a chunking strategy or a longer-context model becomes the live trade-off.

**Re-spike trigger:** T0.2.x connector ingestion task kickoff.

**Primary candidate for re-spike:** `nomic-embed-text-v1.5`. 8192-token native context, Matryoshka truncation 768→512→256→128→64 (preserves the LanceDB 384-or-similar schema with minimal migration), Apache-2.0. Trade-offs to evaluate: ~3× param count (latency budget), mandatory `search_query:` / `search_document:` prefix injection (API redesign), no first-party ONNX (would weaken ADR-019 chain).

**NOT a present-day switch trigger.** V0.1 manual-entry workflow stays well under 512 tokens for typical memories.

### 6.2 — Annual model-currency re-check

**Watch-point:** bge-small-en-v1.5 was released September 2023 — ~2.5 years old as of this spike. SOTA in small-class English embedding has been shifting roughly yearly. Don't assume the lock holds across multi-year intervals.

**Re-run cadence:** Re-run this spike at **V0.2 release** and again **before V1.0 GA**. The same matrix structure applies; the candidate list will need refresh (e.g., Jina v5-small, Qwen3-embedding-small, whatever sits at the head of MTEB at the time).

**Don't assume continuity.** Even if no candidate dethrones bge-small at re-check, the verification is what makes the lock trustworthy.

### 6.3 — Future-swap evaluation criteria (locked)

If a future model swap is ever evaluated, these are the criteria — applied in order, with the first three as hard requirements:

1. **License clean for commercial closed-source distribution.** Hard pass/fail.
2. **First-party ONNX export available** (from the model author, not community-converted). Hard requirement for ADR-019 trust posture; community ONNX requires its own re-verification path.
3. **Prefix-injection compatibility with `BgeSmallProvider`'s API surface** — OR explicit `BgeSmallProvider` API-surface amendment + downstream propagation through `vault-retrieval`. Models requiring mandatory prefixes (e5 family, nomic family, etc.) trigger an explicit amendment phase before the swap, not a silent integration.
4. **MTEB-R parity or improvement** vs the 51.68 NDCG@10 baseline. Sub-baseline retrieval requires explicit justification.
5. **≤ 512 MB ONNX disk size** for installer-fatness budget. Larger requires ADR-amendment + V1.0 download-progress UX revisit per BRD §6 V1.0 installer concerns.

## 7. Reconciliation note (cross-pass)

This spike was run twice independently (Shahbaz pass + Claude pass). Both landed (a). Convergence points and divergences:

**Convergences (no action, alignment confirmed):**
- License MIT, commercial closed-source clean — verified independently from BAAI's official model card.
- First-party ONNX in BAAI's repo as the load-bearing differentiator vs all alternatives.
- All locked artefacts stand: ADR-019 / ADR-020 / ADR-022 / BRD §5.3.

**Divergence resolved in favour of the more honest framing:**
- The Claude pass's initial forward-pointer named arctic-embed-s as the cleanest drop-in substitute. The Shahbaz pass correctly flagged that arctic-embed-s is built on the e5-small base and likely inherits e5's mandatory prefix requirement — which would NOT be a drop-in. **The corrected forward-pointer is recorded in §6.3 criterion 3 as a first-class evaluation criterion.**

**Numerical pin:**
- bge-small-en-v1.5 retrieval = **51.68 NDCG@10** (vs arctic-s 51.98; both per Snowflake's own model card comparison). Use this number going forward where the score is referenced.

**Empirical bonus confirmation (2026-05-03):** the same CI green run that became the first data point for CLAUDE.md candidate #8 (run `25281167500` on commit `ca4c190`) also re-verified ADR-020's SHA-256 integrity chain end-to-end against BAAI's currently-served `model.onnx` — `setup-dev-env.sh`'s `EXPECTED_MODEL_SHA256` verification passed cleanly on the Linux runner. Independent of the license/research conclusions in this spike, **the paired-files integrity scheme is intact as of 2026-05-03 and the canonical hash in `crates/vault-embedding/src/integrity.rs` matches reality.**
